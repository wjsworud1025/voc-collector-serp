================================================================================
  1. 프로그램 소개
================================================================================

VOC Collector(SerpApi)는 전 세계 24개국 소비자 리뷰·후기·평가를 AI가 자동으로
수집하고 분석하여 PDF 보고서로 만들어주는 데스크톱 앱입니다.

[ 핵심 기능 ]
  - AI 조사계획 자동 수립 (Google Gemini)
  - SerpApi를 통한 글로벌 검색 (Google / Naver / Yahoo Japan / Yandex /
    Baidu / Amazon / eBay / Walmart / Home Depot / Google Shopping)
  - 제품 리뷰 페이지 자동 크롤링 및 AI 요약
  - 긍정 / 부정 / 중립 감성 자동 분류
  - 리뷰 검토 및 승인/반려 기능
  - 4가지 형식의 PDF 보고서 자동 생성


================================================================================
  2. 시스템 요구사항
================================================================================

  - 운영체제   : Windows 10 / 11 (64bit)
  - 메모리     : 4GB RAM 이상 권장
  - 디스크     : 설치 후 약 250MB 사용
  - 인터넷     : 수집 작업 중 필수 (API 호출)
  - API 키     : Google Gemini API 키, SerpApi 키 (둘 다 필수)


================================================================================
  3. 설치 방법
================================================================================

  1) 설치 파일 실행
     "VOC Collector (SerpApi)_1.5.0_x64-setup.exe" 를 더블클릭합니다.

  2) 설치 마법사 진행
     화면 안내에 따라 [다음] → [설치] → [완료]를 클릭합니다.
     기본 설치 경로: C:\Program Files\VOC Collector (SerpApi)

  3) 바탕화면 아이콘 또는 시작 메뉴에서 앱을 실행합니다.

  ※ 백신 프로그램이 설치를 차단할 경우, 허용 후 다시 시도하세요.
  ※ 관리자 권한이 필요할 수 있습니다.


================================================================================
  4. 초기 설정 (API 키)
================================================================================

앱을 처음 실행하면 Settings(설정) 화면이 표시됩니다.

  [ 필수 입력 항목 ]

  ① Gemini API Key
     - 발급처: https://aistudio.google.com/apikey
     - AI 조사계획 수립, 리뷰 내용 분석에 사용됩니다.
     - 무료 티어 제공 (분당 요청 제한 있음)

  ② SerpApi Key
     - 발급처: https://serpapi.com/
     - 전 세계 검색 결과·리뷰 수집에 사용됩니다.
     - 월 250회 무료 / 유료 플랜으로 확장 가능

  [ 선택 입력 항목 ]

  ③ Gemini Model (기본값: gemini-2.5-flash)
     - 다른 Gemini 모델로 변경 가능
     - 예) gemini-2.5-pro (더 정확하지만 느림)

  입력 완료 후 [저장] 버튼을 클릭하세요.
  저장 후 앱을 재시작하면 설정이 적용됩니다.


================================================================================
  5. 기본 사용 방법
================================================================================

  STEP 1. 새 프로젝트 생성
  ─────────────────────────
  홈 화면에서 [+ 새 프로젝트] 클릭
  → 프로젝트명 입력 (예: "에어프라이어 미국 시장 조사")

  STEP 2. 조사 요청 입력
  ─────────────────────────
  입력창에 조사하고 싶은 내용을 자유롭게 적습니다.
  → [AI 조사계획 생성] 버튼 클릭
  → AI가 최적의 검색 전략을 자동 수립합니다. (수초~수십초 소요)

  STEP 3. 데이터 수집
  ─────────────────────────
  생성된 계획 확인 후 [수집 시작] 버튼 클릭
  → 검색 → 크롤링 → AI 분석 순서로 자동 진행
  → 진행 상황은 우측 타임라인에서 확인

  STEP 4. 리뷰 검토
  ─────────────────────────
  수집 완료 후 [리뷰 검토] 탭에서 결과를 확인합니다.
  → 각 리뷰의 [승인] / [반려] 버튼으로 보고서 포함 여부 결정

  STEP 5. 보고서 생성
  ─────────────────────────
  [보고서] 탭에서 보고서 형식 선택 후 [PDF 생성] 클릭
  → 생성된 PDF는 문서 폴더 > VOC Reports에 저장됩니다.


================================================================================
  6. 좋은 질문 예시
================================================================================

  아래는 AI가 정확하고 풍부한 조사계획을 세울 수 있는 질문 패턴입니다.

  ────────────────────────────────────────────────────────────────────────────
  ● 국가 + 제품 + 수집 목적을 함께 명시
  ────────────────────────────────────────────────────────────────────────────

  ✅ "미국 소비자들이 에어프라이어에 대해 어떤 불만을 가지고 있는지 알고 싶어"
     → 미국 시장, 에어프라이어, 불만 중심 수집 — 명확

  ✅ "일본 시장에서 무선청소기 구매 결정 요인을 조사해줘"
     → Yahoo Japan + Amazon Japan 활용, 일본어 리뷰 수집

  ✅ "한국 20~30대 여성이 선크림에 대해 어떤 포인트를 중요하게 생각하는지"
     → 네이버 블로그·카페 + 화해 타겟, 성분/사용감 등 키워드 구성

  ✅ "독일 소비자가 전기차 충전기에 대해 남긴 리뷰를 Amazon DE, eBay DE 중심으로"
     → 플랫폼까지 직접 지정한 구체적인 요청

  ✅ "러시아에서 인기 있는 로봇청소기 브랜드와 소비자 평가는?"
     → Yandex 검색 엔진 자동 선택

  ✅ "중국 샤오미 공기청정기 사용자 실제 후기를 알고 싶어"
     → Baidu 검색 엔진 자동 선택, 중국어 수집

  ────────────────────────────────────────────────────────────────────────────
  ● 복수 국가 비교 조사
  ────────────────────────────────────────────────────────────────────────────

  ✅ "미국, 독일, 일본 3개국에서 식기세척기 소비자 반응을 비교해줘"
     → 국가별로 Google/Amazon/Yahoo Japan 각각 활용

  ✅ "동남아시아(베트남, 인도네시아, 태국) 스킨케어 제품 구매 후기 수집"
     → 각 국가 현지 언어로 검색 자동 설정

  ────────────────────────────────────────────────────────────────────────────
  ● 특정 플랫폼/채널 지정
  ────────────────────────────────────────────────────────────────────────────

  ✅ "Amazon US에서 커피머신 리뷰 중 별점 1~3점대 불만 사항을 집중 수집해줘"
     → 부정적 리뷰 중심, Amazon 엔진 직접 활용

  ✅ "eBay에서 중고 카메라 구매자 후기를 바탕으로 품질 이슈를 파악해줘"
     → eBay 검색 엔진 활용

  ✅ "Walmart 구매자들이 가정용 선풍기에 대해 어떤 얘기를 하는지 알고 싶어"
     → Walmart 엔진 자동 선택


================================================================================
  7. 좋지 않은 질문 예시
================================================================================

  아래 패턴은 AI가 조사 방향을 잡기 어렵거나 품질이 낮아질 수 있습니다.

  ────────────────────────────────────────────────────────────────────────────
  ● 너무 짧거나 모호한 질문
  ────────────────────────────────────────────────────────────────────────────

  ✗ "청소기"
     → 어느 나라? 어떤 목적? 어느 브랜드? — AI가 추측으로 계획을 세워야 함
     → 개선: "미국 시장 무선청소기 소비자 불만 조사"

  ✗ "리뷰 수집해줘"
     → 무엇에 대한 리뷰인지 불명확
     → 개선: "한국 시장 공기청정기 구매자 리뷰 수집"

  ✗ "제품 조사"
     → 제품명, 국가, 목적이 모두 빠짐
     → 개선: "일본 소비자의 전기밥솥 불만 포인트 조사"

  ────────────────────────────────────────────────────────────────────────────
  ● 범위가 지나치게 넓은 질문
  ────────────────────────────────────────────────────────────────────────────

  ✗ "전 세계 모든 가전제품 리뷰 수집해줘"
     → API 한도 초과, 수집 시간 과다, 결과 품질 저하
     → 개선: "미국/독일/일본 3개국 로봇청소기 리뷰 수집"

  ✗ "모든 나라에서 화장품 후기 다 긁어줘"
     → 개선: "동남아 3개국 자외선차단제 소비자 반응 조사"

  ────────────────────────────────────────────────────────────────────────────
  ● 제품·서비스가 아닌 주제
  ────────────────────────────────────────────────────────────────────────────

  ✗ "대통령 선거 여론 조사해줘"
     → 이 프로그램은 소비자 제품 VOC 수집 전용입니다.

  ✗ "주식 종목 추천해줘"
     → 투자/금융 정보 수집은 지원하지 않습니다.

  ✗ "뉴스 기사 크롤링해줘"
     → 소비자 리뷰가 아닌 뉴스 수집은 설계 범위 외입니다.

  ────────────────────────────────────────────────────────────────────────────
  ● 비현실적인 기대
  ────────────────────────────────────────────────────────────────────────────

  ✗ "리뷰 10만 건 수집해줘"
     → SerpApi 무료 플랜은 월 100회 제한 (건당 최대 10~20개 결과)
     → 실제 수집량: 프로젝트당 수십~수백 건 수준

  ✗ "오늘 올라온 실시간 리뷰만"
     → 날짜 필터는 검색 엔진마다 지원 여부가 다르며 보장되지 않습니다.

================================================================================
  9. PDF 보고서 종류
================================================================================

  1) 표준 보고서 (Standard)
     - 수집된 VOC 전체 목록
     - 감성 분포 (긍정/부정/중립)
     - 플랫폼별 통계

  2) 경영진 요약 (Executive)
     - 핵심 긍정/부정 의견 TOP 3
     - 한 페이지 요약 형식

  3) 상세 보고서 (Detailed)
     - 플랫폼별 세부 통계 + 수집 방법 분포
     - 주요 토픽 빈도 분석

  4) 프리미엄 인사이트 (Premium)
     - Gemini AI가 생성한 전략적 인사이트 포함
     - 시사점, 개선 제안, 리스크 분석
     ※ [프리미엄 분석 생성] 버튼을 먼저 눌러야 합니다.

  10. 자주 묻는 질문 (FAQ)
================================================================================

  Q. "조사계획 수립 실패: 404 NOT_FOUND" 오류가 납니다.
  A. Settings에서 Gemini API Key가 올바르게 입력되었는지 확인하세요.
     Gemini Model이 잘못된 경우에도 발생합니다. (기본값: gemini-2.5-flash)

  Q. 수집이 아무것도 안 됩니다.
  A. SerpApi Key 확인, 월 사용 한도 초과 여부를 serpapi.com 대시보드에서 확인하세요.

  Q. 일본어/중국어/러시아어 글자가 깨집니다.
  A. Windows 시스템 폰트(맑은 고딕, Yu Gothic, Tahoma)가 사용됩니다.
     Windows 10/11이라면 자동으로 지원됩니다.

  Q. PDF에 글자가 □□□로 나옵니다.
  A. Windows 폰트 디렉토리(C:\Windows\Fonts)에 malgun.ttf, tahoma.ttf가
     있는지 확인하세요. 없다면 Windows 복구를 시도하세요.

  Q. 수집 속도가 너무 느립니다.
  A. SerpApi 유료 플랜으로 업그레이드하거나, 수집할 플랫폼 수를 줄여주세요.
     Gemini API 무료 티어의 분당 요청 제한도 원인이 될 수 있습니다.

  Q. 보고서가 저장되는 위치는 어디인가요?
  A. 기본 저장 경로: C:\Users\[사용자명]\Documents\VOC Reports\

  Q. 앱을 삭제하려면?
  A. Windows 설정 > 앱 > "VOC Collector (SerpApi)"를 선택 후 제거하세요.
     수집 데이터(DB 파일)는 별도로 삭제해야 합니다.
     데이터 위치: C:\Users\[사용자명]\AppData\Local\com.voc-collector-serp.app\
